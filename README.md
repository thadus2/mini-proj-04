##  📖 1. 프로젝트 이름 및 서비스 개요


📚 KT AIVLE School 4차 미니프로젝트 - 📚 **<당신의 이야기에 완벽한 옷을 입히세요 - AI 북스>**

> 누구나 작가가 되어 자유롭게 글을 집필하고, AI 이미지 생성 기술로 내 이야기가 담긴 단 하나의 표지를 완성하는 도서 관리 시스템

국내 최대 독서 플랫폼을 지향하는 **\"AI Book\"**은 창작자의 감성을 온전히 담아내는 자유 창작 플랫폼입니다. 

단순히 글을 쓰고 공개하는 것을 넘어, **작가의 이야기와 시선이 그대로 표지에 닿을 수 있도록 설계**되었습니다. 전문 디자이너나 표지 기획자가 없어도, AI 이미지 생성 기술이 창작자의 가장 가까운 조력자가 되어 글의 감성을 시각적인 표지로 직관적으로 구현해 냅니다.

## ✨ 3. 프로젝트 목표


1. **AI 기반 도서 표지 자동 생성**
   * 창작자가 작성한 글의 내용과 감성을 바탕으로 맞춤형 도서 표지를 자동 생성합니다.

   * 진입 장벽을 낮춰 누구나 쉽게 작가가 되어 자신의 작품을 제작하고 공개할 수 있습니다.

## 💾 6. 도서 데이터 구조

`db.json` 파일의 전체적인 데이터 스키마 구조입니다. 가상 API 서버(json-server)와 통신 시 아래와 같은 정형화된 JSON 포맷을 활용합니다.

```json
{
  "books": [
    {
      "id": 1,
      "title": "별빛 아래의 서점",
      "author": "홍길동",
      "genre": "미스터리/드라마",
      "content": "줄거리 요약 내용이 이곳에 저장됩니다...",
      "likes": 45,
      "views": 13,
      "summary": "책 목록 페이지에 노출될 핵심 3문장 요약 정보입니다.",
      "publish": "kt출판사",
      "coverImageUrl": "[https://openai.api/generated-image-url.png](https://openai.api/generated-image-url.png)",
      "createdAt": "2026-04-24T09:00:00.000Z",
      "updatedAt": "2026-04-24T09:00:00.000Z"
    }
  ]
}
```

### 🔍 필드 상세 설명 (Field Details)
* **`id`** (Number) : 도서 고유 등록 아이디 (자동 생성)
* **`title`** (String) : 도서 제목
* **`author`** (String) : 저자명
* **`genre`** (String) : 도서 장르 (예: 미스터리/드라마, 소설 등)
* **`content`** (String) : 전체 줄거리 및 본문 내용
* **`likes`** (Number) : 좋아요 수 (추천 수)
* **`views`** (Number) : 도서 상세 페이지 조회수 (방문자 수)
* **`summary`** (String) : 메인 목록 화면에 노출될 3문장 이내의 요약문
* **`publish`** (String) : 출판사명
* **`coverImageUrl`** (String) : OpenAI DALL-E API를 통해 자동 생성된 고유 표지 이미지 URL
* **`createdAt`** (String) : 최초 도서 등록 일시 (ISO 8601 포맷)
* **`updatedAt`** (String) : 최근 도서 정보 수정 일시 (ISO 8601 포맷)

## 👔 7. 시작 가이드 (Getting Started)

본 프로젝트를 로컬 환경에서 설치하고 실행하는 방법입니다. 원활한 구동을 위해 아래의 단계를 순서대로 진행해 주세요.

### 🛠️ 1. React & Vite 클라이언트 빌드 및 실행
터미널(CMD 또는 Terminal)을 열고 프로젝트를 다운로드한 후 프론트엔드 개발 서버를 구동합니다.

```bash
# 1. 원격 저장소 복제 (Clone)
git clone [https://github.com/thadus2/mini-proj-04.git](https://github.com/thadus2/mini-proj-04.git)

# 2. 프로젝트 폴더로 이동
cd mini-proj-04

# 3. 의존성 패키지 설치
npm install

# 4. 로컬 개발 서버 실행
npm run dev
```
> 💡 서버 가동 후 터미널 창에 출력되는 로컬 주소(예: `http://localhost:5173`)를 클릭하면 도서관리시스템 포털에 접속할 수 있습니다.

---

### 📦 2. JSON 서버 (Mock DB) 준비 및 가동
도서 데이터 처리를 위한 가상 API 서버를 실행합니다. **이전 터미널 창은 그대로 띄워둔 상태에서, 새로운 터미널 창을 열고** 아래 명령어를 입력하세요.

```bash
# 1. json-server 전역 설치 (최초 1회만 진행)
npm install -g json-server

# 2. 새 터미널 창에서 프로젝트 폴더로 이동
cd mini-proj-04

# 3. Mock DB 서버 가동 (3000번 포트 지정)
npx json-server@0.17.4 --watch mockdb.json --port 3000
```

---

### 🔄 3. GitHub 최신 동기화 가이드
원격 저장소의 최신 업데이트 내역을 로컬 작업 공간에 강제로 갱신하고자 할 때 사용합니다.

```bash
# 원격 저장소의 모든 브랜치 정보 가져오기
git fetch --all

# 로컬 상태를 원격 main 브랜치의 최신 상태로 강제 초기화
git reset --hard origin/main
```
> ⚠️ **주의 사항:** `git reset --hard` 명령어를 실행하면 현재 로컬에서 수정 중이던 미커밋 파일이 모두 유실될 수 있습니다. 중요한 작업 내역이 있다면 실행 전에 반드시 별도로 백업해 두시기 바랍니다.
